### 3. 사용자 시나리오

사용자가 실제로 이 페이지에서 어떤 흐름으로 행동하는지 순서대로 작성합니다.

</aside>

**기본 흐름 (작성)**

1. 사용자가 게시글 상세 페이지에서 댓글 목록으로 내린다.
2. 로그인 사용자는 댓글을 작성할 수 있고 사용자 태그(자동완성, 추천기능으로 간편하게)가 가능하다.
3. 글자는 최대 500글자로 작성하게 설정.
4. 댓글을 빈칸으로 작성한 경우 1글자 이상 작성해야 합니다. 토스트 메시지 남기기

**성공 시 흐름 (작성 성공)**

1. 게시글에서 댓글 등록 (엔터 or 버튼누르기)
2. POST/api/v1/posts/{post_id}/comments 호출 성공 (201).
3. 댓글 목록 화면에 등록한 댓글이 추가됩니다.

**실패 시 흐름**

1. (서버 오류인 경우) 에러 토스트 "요청에 실패했습니다." 표시 후 폼 유지.
2. "401: {
   "error_detail": "자격 인증 데이터가 제공되지 않았습니다."
   } 로그인이 되어있는지 안되어있는지, 로그인이 만료되면 토스트 메시지 호출 후 로그인 화면으로 이동
3. 댓글창에 1글자도 안적고 작성을 누른 경우 400: {
   "error_detail": {
   "content": [
   "댓글은 공백상태로 등록이 안됩니다."
   ]
   }
   } 토스트 메시지 남기고 폼 유지(원래 화면)
4. 404: {
   "error_detail": "게시글이 없습니다."
   } 게시글 목록 페이지로 이동

### 4. 요구사항 정리

기획서/피그마/회의 내용을 바탕으로 실제 구현해야 하는 요구사항만 적습니다.

**필수 요구사항**

- [ ] 댓글목록에 등록버튼 구현 (로그인 시에만 보임)
- [ ] 댓글 작성은 최대 500자까지 가능하며 타 유저를 태그할 수 있어야 함.
- [ ] 모든 오즈코딩스쿨 이용자 중 로그인 된 회원은 게시글에 댓글 작성 중 타 유저를 태그할 수 있습니다.
- [ ] 유저 태깅은 @ 문자를 입력하여 진행할 수 있으며 닉네임으로 태그할 수 있습니다.

### 댓글 작성 (POST)

사용자가 입력창에 내용을 쓰고 '등록' 버튼을 눌렀을 때 데이터를 서버에 저장하는 기능입니다.

- **Method:** `POST`
- **Endpoint :** api/v1/posts/{post_id}/comments
- **보낼 데이터 (Body):**
  - `content`: 댓글 내용
- **특이사항:** 성공 시 화면을 새로고침하거나, 방금 추가된 데이터를 리스트에 즉시 반영(Optimistic Update 등)하는 처리가 필요
  모든 오즈코딩스쿨 이용자 중 로그인 된 회원은 게시글에 댓글을 작성할 수 있습니다.

**선택 요구사항**

없음.

### 6. API 연동 정리

가장 중요한 영역입니다. AI가 구현하려면 **어떤 API를 언제 왜 호출하는지**가 명확해야 합니다.

</aside>

**API 목록**

| 기능      | 메서드 | 엔드포인트                            | 호출 시점        | 설명                                                                     |
| --------- | ------ | ------------------------------------- | ---------------- | ------------------------------------------------------------------------ |
| 댓글 작성 | POST   | /api/v1/posts/{post_id}/comments      | 등록 버튼 클릭시 | 등록버튼을 눌러서 사용자의 텍스트 데이터를 **DB에 영구적으로 저장한다.** |
| 유저 태그 | GET    | /api/v1/users/search?nickname={query} | @사용자 태그시   | 태그할 유저 목록을 자동완성으로 보여주기 위함                            |

**API 상세 명세**

<aside>

- 유저 태그 시 api 작동
  - **기능:** 유저 태그
  - **메서드:** GET
  - **엔드포인트:** `/api/v1/users/search?nickname={query}`
  - **인증:** 필요 (`Bearer Token`)
  - **파라미터 타입:**
    - **쿼리파라미터 :** nickname
  - **Responese Body**:
  ```json
  "200: {
        "results": [
            {
                "id": 2,
                "nickname": "testuser2",
                "profile_img_url": "https://example.com"
            },
            {
                "id": 3,
                "nickname": "testuser3",
                "profile_img_url": "https://example.com"
            }
        ]
    }"
  ```

  - **실패 응답 예시**:
  ```json
   "401: {
    "error_detail": "자격 인증 데이터가 제공되지 않았습니다"
  }"
  ```

</aside>

<aside>

- 댓글 작성 시 api 작동
  - **기능:** 댓글 작성
  - **메서드:** `POST`
  - **엔드포인트:** `/api/v1/posts/{post_id}/comments`
  - **인증:** 필요 (`Bearer Token`)
  - **파라미터 타입:**
    - **Path Variable:** `post_id` (대상 게시글 ID)
    - **Request Body:** `{"content": "string"}`
  - **특이사항:** 사용자 태그 정보(`@nickname`)는 `content` 문자열에 포함하여 전송함.
  - **Request Body**:
  ```json
  "{
      "content": "@testuser2 를 태그한 댓글 작성 테스트"
  }"
  ```

  - **성공 응답 예시**:
  ```json
  "201: {
      "detail": "댓글이 등록되었습니다."
  }"
  ```

  - **실패 응답 예시**:
  ```json
    "400: {
      "error_detail": {
        "content": [
          "이 필드는 필수 항목입니다."
        ]
      }
  }

  401: {
      "error_detail": "자격 인증 데이터가 제공되지 않았습니다."
  }

  404: {
      "error_detail": "해당 게시글을 찾을 수 없습니다."
  }"
  ```

</aside>

### 7. 프론트엔드 동작 명세

API를 받아서 화면에서 어떻게 동작해야 하는지 적습니다.

</aside>

**사용자 액션**.

- 등록 버튼 클릭 시 (로그인 된 상태여야 함):
- 유효성 검사 → API 요청 전송 → 응답 처리 (밑에 어떤식으로 작동되는지 적혀있음)
- **포커스 처리:** 작성 완료 후 입력창이 비워지면 자동으로 입력창에 다시 포커스가 가지 않도록 처리 (중복 방지).

### **유효성 검사 (Validation)**

- **내용 확인:** `content`가 비어있거나 공백만 있는지 확인합니다.
  - _비어있을 경우:_ "댓글 내용을 입력해주세요."라는 토스트 메시지를 띄우고 폼 유지(원래 상태).
- **글자수 제한:** (선택 사항) 서버에 보내기 전 프론트엔드에서 최대 글자수(예: 500자)를 체크합니다.

### **API 요청 전송 (Request)**

- **버튼 비활성화:** 중복 클릭으로 인한 다중 등록을 방지하기 위해 '등록' 버튼을 `disabled` 상태로 만듭니다.
- **API 호출:** \* **Method:** `POST`
  - **URL:** `/api/v1/posts/{post_id}/comments`
  - **Body:** `{ "content": "..." }`

### **응답 처리 (Response)**

**A. 성공 시 (HTTP 201 Created)**

1. **입력창 초기화:** 작성했던 댓글 내용을 지웁니다.
2. **화면 업데이트:** **\*(새로고침 없이):** 서버에서 받은 응답 데이터(새 댓글 객체)를 기존 댓글 리스트 배열 맨 앞이나 맨 뒤에 즉시 추가(Push/Unshift)합니다.
3. **버튼 활성화:** 버튼을 다시 클릭 가능한 상태로 되돌립니다.

**B. 실패 시 (HTTP 4xx, 5xx)**

1. **에러 핸들링:**
   - **401 Unauthorized:** "로그인이 필요한 서비스입니다." 알림 후 로그인 페이지로 이동.
   - **404 Not Found:** "존재하지 않는 게시글입니다." 알림.
   - **5xx:** "서버 점검 중입니다. 잠시 후 다시 시도해주세요." 알림.
2. **버튼 활성화:** 실패하더라도 사용자가 내용을 수정해서 다시 시도할 수 있도록 버튼을 다시 활성화합니다.
