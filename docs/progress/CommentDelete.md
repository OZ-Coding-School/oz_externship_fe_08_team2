### 3. 사용자 시나리오

사용자가 실제로 이 페이지에서 어떤 흐름으로 행동하는지 순서대로 작성합니다.

</aside>

**기본 흐름 (작성)**

1. 사용자가 게시물 상세 페이지에서 댓글 목록을 본다.
2. 내가 작성한 댓글이 있는지 확인
3. 내가 작성한 댓글 왼쪽부분 삭제버튼 클릭

**성공 시 흐름 (작성 성공)**

1. 삭제버튼을 클릭한다
2. 삭제시 DELETE /api/v1/posts/{post_id}/comments/{comment_id} 호출 성공 (200).
3. 성공 토스트 메시지 “댓글이 삭제되었습니다.” 표시

**실패 시 흐름**

1. (서버 오류인 경우) 에러 토스트 "요청에 실패했습니다." 표시 후 게시물 상세페이지 폼으로 돌아가기
2. 삭제하려고 버튼을 눌렀는데 게시글이 삭제된 경우"404: {
   "error_detail": ""해당 게시글을 찾을 수 없습니다."
   }"
   이 경우는 토스트 후 게시글 목록 페이지로 이동 합니다.
3. 댓글이 이미 지워진 경우(관리자에 의해?) “404: { "error_detail": "해당 댓글을 찾을 수 없습니다." }"
   이 경우는 토스트 후 원래 폼(게시글 상세) 에서 새로고침 합니다.
4. 401: {
   "error_detail": "자격 인증 데이터가 제공되지 않았습니다."
   }
   이 경우는 토스트 후 로그인 페이지로 이동.

### 댓글 삭제 (DELETE)

- [ ] '삭제' 버튼을 눌렀을 때 특정 댓글을 DB에서 지우는 기능입니다.
- **Method:** `DELETE`
- **Endpoint :** api/v1/posts/{post_id}/comments/{comment_id}
- **필요한 데이터:** 삭제하려는 댓글의 고유 **ID값**, Access Token, post_id(어디서)
- **보안 사항:** **다중 계층 검증:** \* **(Client)** 작성자 본인이 아닌 경우 삭제 버튼 비활성화 및 UI 노출 제한.

### 6. API 연동 정리

가장 중요한 영역입니다. AI가 구현하려면 **어떤 API를 언제 왜 호출하는지**가 명확해야 합니다.

</aside>

**API 목록**

| 기능      | 메서드 | 엔드포인트                                   | 호출 시점         | 설명                                                         |
| --------- | ------ | -------------------------------------------- | ----------------- | ------------------------------------------------------------ |
| 댓글 삭제 | DELETE | api/v1/posts/{post_id}/comments/{comment_id} | 삭제 버튼 클릭 시 | 사용자가 본인이 작성한 특정 댓글을 영구 삭제하고 DB에 반영함 |

**API 상세 명세**

<aside>

- **API 템플릿**
  - **기능:** 댓글 삭제
  - **메서드:** `DELETE`
  - **엔드포인트:** `/api/v1/posts/{post_id}/comments/{comment_id}`
  - **인증:** 필요 (`Bearer Token`)
  - **파라미터 타입:**
    - **Path Variable:** `post_id`, `comment_id`
  - **특이사항:** 서버에서 토큰의 `user_id`와 댓글의 `author_id`를 비교하여 403 에러 처리 필수.
  - **성공 응답 예시**:
  ```json
  "200: {
      "detail": "댓글이 삭제되었습니다."
  }"
  ```

  - **실패 응답 예시**:
  ```json
   401: {
      "error_detail": "자격 인증 데이터가 제공되지 않았습니다."
  }

  403: {
      "error_detail": "권한이 없습니다."
  }

  404: {
      "error_detail": "해당 댓글을 찾을 수 없습니다."
  }

  ```

</aside>

### 7. 프론트엔드 동작 명세

API를 받아서 화면에서 어떻게 동작해야 하는지 적습니다.

</aside>

**사용자 액션**

- 삭제 버튼 클릭 시: 사용자가 삭제를 누르면, 해당 댓글의 `comment_id`와 `post_id`를 사용하여 `DELETE` API를 호출합니다. 이때 헤더에 사용자 인증 토큰(JWT)을 함께 전송합니다.
- 요청이 처리되는 동안 삭제 버튼을 비활성화(disabled)하여 중복 클릭을 방지합니다.
- **성공(200 OK):** 화면의 댓글 목록에서 해당 댓글을 즉시 제거(UI 업데이트)하고, "댓글이 삭제되었습니다."라는 안내 메시지(Toast 등)를 노출합니다
- **실패(4xx, 5xx):** 에러 메시지에 따른 알림창을 띄우고(예: "본인만 삭제 가능합니다"), 화면 상태를 유지합니다.
