<aside>

### 3. 사용자 시나리오

</aside>

**기본 흐름 (작성)**

1. 로그인한 사용자가 게시글 상세 페이지에서 댓글 목록으로 스크롤한다.
2. 입력창에 댓글을 작성한다 (최대 500자).
3. 등록 버튼을 클릭한다.

**성공 시 흐름**

1. 등록 버튼 클릭 → `POST /api/v1/posts/{post_id}/comments` 호출 (201).
2. 입력창이 초기화된다.
3. 댓글 목록이 자동으로 갱신되어 새 댓글이 표시된다.

**실패 시 흐름**

1. 빈 칸으로 제출 → 버튼이 비활성화되어 요청 자체가 차단됨.
2. 400 (서버 유효성 실패) → "댓글 내용을 입력해주세요." 토스트, 폼 유지.
3. 401 (인증 만료) → "로그인이 필요합니다." 토스트 후 로그인 페이지로 이동.
4. 404 (게시글 없음) → "존재하지 않는 게시글입니다." 토스트 후 게시글 목록으로 이동.
5. 5xx (서버 오류) → "요청에 실패했습니다. 잠시 후 다시 시도해주세요." 토스트, 폼 유지.

### 6. API 연동 정리

</aside>

**API 목록**

| 기능      | 메서드 | 엔드포인트                       | 호출 시점         | 설명                    |
| --------- | ------ | -------------------------------- | ----------------- | ----------------------- |
| 댓글 작성 | POST   | /api/v1/posts/{post_id}/comments | 등록 버튼 클릭 시 | 댓글 내용을 서버에 저장 |

**API 상세 명세**

<aside>

- **기능:** 댓글 작성
- **메서드:** `POST`
- **엔드포인트:** `/api/v1/posts/{post_id}/comments`
- **인증:** 필요 (`Bearer Token`)
- **Request Body:**

```json
{
  "content": "댓글 내용"
}
```

- **성공 응답 (201):**

```json
{
  "detail": "댓글이 등록되었습니다."
}
```

- **실패 응답:**

```json
"400: { \"error_detail\": { \"content\": [\"이 필드는 필수 항목입니다.\"] } }
401: { \"error_detail\": \"자격 인증 데이터가 제공되지 않았습니다.\" }
404: { \"error_detail\": \"해당 게시글을 찾을 수 없습니다.\" }"
```

</aside>

### 7. 프론트엔드 동작 명세

</aside>

**유효성 검사**

- `content`가 비어있거나 공백만 있을 경우 버튼 `disabled` → 요청 차단.

**API 요청**

- 등록 버튼 클릭 시 버튼 `disabled` (중복 요청 방지).
- `POST /api/v1/posts/{post_id}/comments` 호출.

**응답 처리**

- 201 성공: 입력창 초기화 → `invalidateQueries`로 댓글 목록 자동 갱신.
- 400: "댓글 내용을 입력해주세요." 토스트, 폼 유지.
- 401: "로그인이 필요합니다." 토스트 후 로그인 페이지로 이동.
- 404: "존재하지 않는 게시글입니다." 토스트 후 게시글 목록으로 이동.
- 5xx: "요청에 실패했습니다. 잠시 후 다시 시도해주세요." 토스트, 폼 유지.
